# PPT 大纲：试验大纲智能审查系统

## 第 1 页：什么是试验大纲审查，系统解决什么问题

### 标题
试验大纲智能审查：从人工经验检查到可追溯的智能审查

### 页面主旨
试验大纲审查是对试验方案类文档进行合规性、完整性和一致性检查。当前系统的核心价值，是把 DO-160G 检查单和专家审查经验转化为可执行、可定位、可复核的智能审查流程。

### 1. 什么是试验大纲审查

- 审查对象：试验大纲、试验方案等 Word 文档。
- 审查目标：判断文档是否满足试验要求、工程规范和检查单要求。
- 典型审查内容：
  - 试验项目是否覆盖完整；
  - 试验条件、设备/仪器、判据、步骤是否明确；
  - 记录要求、数据要求、结论要求是否清晰；
  - 章节之间的术语、参数、名称、编号是否一致；
  - 是否存在遗漏、矛盾、表达不清或证据不足的问题。

### 2. 当前人工审查面临的问题

- 规则多、内容细：DO-160G 检查单和试验大纲要求项多，人工逐项核对耗时长。
- 文档篇幅长：试验大纲通常按章节展开，问题分散在不同位置，人工定位成本高。
- 依赖专家经验：不同审查人员对规则理解、检查粒度和问题表述可能不一致。
- 结果难追溯：传统审查意见常只有问题描述，缺少对应原文证据和定位信息。
- 复核效率低：人工复核需要重新翻文档、找依据、比对规则，流程不够闭环。

### 3. 当前系统解决的问题

- 将检查单规则结构化：把 DO-160G 检查单转化为系统可执行的规则和检查项。
- 自动完成初审：对上传的 Word 试验大纲进行章节解析、规则匹配和 AI 审查。
- 给出原文证据：每条审查意见尽量关联原文片段，支持问题定位和溯源。
- 支持人工复核：AI 结果可由人工改判、确认或补充备注，形成审查闭环。
- 支持成果导出：审查意见、审查报告和审计日志可导出，便于交付和留档。

### 4. 一句话总结

系统把“专家拿检查单逐条看文档”的工作方式，转化为“规则驱动 + AI 初审 + 原文溯源 + 人工复核 + 结果导出”的智能审查流程。

### 可视化建议

- 左侧：人工审查痛点，突出“耗时、易漏、难追溯、难复核”。
- 右侧：系统闭环能力，突出“规则结构化、自动审查、证据定位、人工复核、成果导出”。

## 第 2 页：系统审查逻辑、规则与模型能力

### 标题
系统如何完成审查：规则驱动、模型判定、证据闭环

### 页面主旨
系统通过“文档解析 → 规则调度 → AI 审查 → 原文溯源 → 人工复核 → 结果导出”的流程，将试验大纲审查任务标准化、自动化。

### 1. 系统输入与审查资源

- 待审查文档：Word 文件，支持 `.doc / .docx`，前端上传限制默认 20MB。
- 审查方式：当前重点支持两套审查管线：
  - 全文逐章审查 CHUNK；
  - 结构化精准审查 SAR。
- 规则输入：
  - 支持 `.md / .json / .txt / .yaml / .yml` 规则文件；
  - 支持 `.xlsx / .xls` Excel 检查单导入。
- AI 模型能力：
  - chat 模型 7 个；
  - embedding 模型 1 个；
  - reranker 模型 1 个。

### 2. 当前规则与场景配置

- 当前按“规则条目”统计：共 40 条规则。
- 当前数据库已有：
  - 规则库：CHUNK 1 个、SAR 1 个；
  - 场景：CHUNK 1 个、RAG 3 个、SAR 1 个。
- 审查场景的作用：选择本次审查启用哪组规则库，决定规则范围和审查重点。

### 3. 系统审查逻辑

1. 文档解析
   - 后端解析 Word 文档；
   - 按章节/结构切分正文；
   - 保留原文来源，用于后续定位和溯源。

2. 规则调度
   - 上传文档后，系统根据审查场景加载规则库；
   - 结合规则元数据、适用章节、关键词、文件夹启停状态分发规则；
   - 将规则匹配到对应章节或原文区域。

3. CHUNK 全文逐章审查
   - 按章节切片；
   - 将章节原文和适用规则送入大模型；
   - 返回结构化 JSON 审查结果；
   - 适合做全文覆盖式、逐章扫描式审查。

4. SAR 结构化精准审查
   - 面向检查项进行精准定位；
   - 通过结构、词法、语义三路路由找到相关原文区域；
   - 进行区域级取证、自适应复核和跨章一致性检查；
   - 更适合复杂规则、清单式检查和证据追溯。

5. 结果输出
   - 生成检查矩阵、问题列表、原文证据和审查日志；
   - 支持人工复核；
   - 支持导出 Excel 审查意见、Word 审查报告和 JSON 审计日志。

### 4. 阶段性成果

- 已实现 CHUNK、SAR 两套审查管线，并完成前后端入口和数据表隔离。
- 已实现规则库、规则文件上传、Excel 检查单导入、场景配置、用户规则分配。
- 审查任务：共 46 个，其中 45 个已完成。
- 已形成完整业务闭环：
  - 上传文档 → 选择场景/模型 → 自动审查 → 原文溯源 → 人工复核 → 导出审查成果。
